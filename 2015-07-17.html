io.js 2.4.0 のリリース、ApigeeがFoundationに参加、Intl WG が開始

io.js 2.4 リリース

今週は、io.js v2.4.0 をリリースしました。詳細は GitHub にて確認できます。

主な変更点

2.4.0

  • src: 新たに --track-heap-objects フラグを追加しました。ヒープのスナップショットのためのヒープオブジェクトをトラックします。 (Bradley Meck) #2135
  • readline: キープレスイベントハンドラーが例外を投げた影響でreplがfreezeするのを修正しました。 (Alex Kocharin) #2107
  • npm: v2.13.0に更新しました。詳細は https://github.com/npm/npm/releases/tag/v2.13.0 を参考にしてください。 (Forrest L Norvell) #2152

既知の問題

現在の完全な既知の問題リストは https://github.com/nodejs/io.js/labels/confirmed-bug を参照してください。

  • beforeExit が実行されている最中の非参照のtimersに起因するいくつかの問題は、まだ解決されていません。詳細は #1264 を参照して下さい。
  • REPL 内のサロゲートペアがターミナルのフリーズを引き起こすことがあります。 #690
  • process.send() はドキュメントに記述されている通り、本来は同期的に実行されるはずですが、1.0.2 にて発生した不具合により非同期的に呼び出されてしまうようになりました。詳細は #760 を、修正については #774 を参照して下さい。
  • dns.setServers() を呼び出している間の DNS クエリが、進行中にアサーションに失敗してプロセスをクラッシュさせることがあります。#894
  • url.resolve は2つの完全なホストを解決しようとした際に url の auth の部分を移動する可能性があります。詳細は #1435 を参照してください。

コミュニティより

  • NodeSource は N|Support を 彼らのブログ記事 にて発表しました。 N|Support は、Node.js 開発者、DevOps、IT運用チームを支援するプレミアムサポートです。N|Support についての詳細を知りたい方は、the solution page を参照してください。
  • Apigee は Node Foundation に参加 しました。
  • joyent/docker-node チームは nodejs/docker チームへと移動しました。結果的に、@chorrell と @dcrudgington は @nodejs/docker チームに参加となりました。
  • Intl WG が開始されました。このWGは、Nodeのサポート・国際化・ローカリゼーションを行います。 Intl WG は、参加者を issueのページ で募集しています。
